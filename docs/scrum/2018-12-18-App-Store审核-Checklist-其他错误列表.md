## 2018-12-18-App-Store审核-Checklist-其他错误列表



| 对应审核条款           | 自检内容                                                     | 说明                                                         | 自检结果 |
| ---------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ | -------- |
| 功能                   | 应用启用iCloud存储功能后，必须遵守iOS数据存储指南，否则将被拒。 | IOS的存储规范：<br />1、用户生成的文档和其他数据，或不能被应用所重建的数据应当保存在<Application_Home>/Documents<br/>目录内。这些数据文件会自动通过iCloud备份<br />2、可以重新下载或者重新创建的数据应当保存在<Application_Home>/Library/Caches<br/>目录内。<br />3、临时需要的数据应该保存在<Application_Home>/tmp<br/>目录内。这些文件不会备份到iCloud里，记住不需要它们时立即删除掉这些文件，这样它们就不会继续浪费用户设备的储存空间了。<br />4、使用“do not back<br/>up”属性，不需要iCloud备份文件（比如需要离线环境使用的文件；该属性能在任何目录下生效），由于这些文件占用设备空间，所以应用需要有一套定期监控与清理这些文件的机制。 |          |
| 功能                   | 不允许通过虚拟货币后虚拟道具的奖励方式，鼓励用户绑定手Q或微信。 | 游戏内鼓励好友绑定帐号时，不得提供虚拟货币虚拟道具等奖励，必须也仅可以通过“额外游戏模式、游戏内容解锁、好友互动”等方式鼓励用户绑定帐号。 |          |
| 功能                   | App有QQ会员福利加成的，请在审核期间屏蔽这个功能，通过后再打开。 | QQ会员福利，苹果内部还在讨论，目前为止还没有定论。           |          |
| 功能                   | 不要出现“公测”、“内测”、“体验”等标识、说明，包括APP链接到的官网。 | 提交App<br/>Store审核的应用名称、安装包等所有地方都不能包含有测试、demo等字样，否则都会被拒。 |          |
| 功能                   | 使用私有API的应用会被拒。                                    | 源码中如果恰好定义了方法与非公开的API重名，会导致被拒；比较常见的是，使用的第三方静态库中，包含了非公开的API。可以通过strings或otool命令来查找相关的API。 |          |
| 功能                   | 不正确的使用iOS的广告标识(IDFA)，将会被拒。                  | 1、源代码中使用了广告标识(IDFA)，但游戏实际并没有广告功能，这是不被允许的。如果你的应用程序不提供广告服务，请检查你的代码<br/>- 包括任何第三方库 -
删除所有实例（包括ASIdentifierManager、advertisingIdentifier、AdSupport.framework）。可以用"nm"工具来检查。<br />2、如果你正计划把广告放在未来的版本，请从应用程序中删除广告标识，直到你已包括广告功能。<br />3、IDFA只能在实现和广告相关功能时使用，不能做其他用途。 |          |
| 功能                   | 公司禁止通过使用企业证书打包更新，而且苹果公司也会拒绝。     | 苹果对“游戏内设置更新按钮，通过企业证书打包更新，绕开App<br/>Store体系”有意见，而且我们公司有严格规定，企业证书打的包（RDM上下载的体验包就是打的企业证书）严禁外传，所以，版本更新要通过打发布证书的方式实现。 |          |
| 功能                   | 使用排行榜的游戏必须加入相关条款说明                         | 我们的UI界面必须有”<br/>极致游戏许可及服务协议”，”隐私政策”和”服务条款”三个按钮，建议这三个按钮加在设置里，外部链接或者游戏内展示内容都可，相关的链接如下：<br />[极致通行证注册许可协议：https://res.jzyx.com/passport/protocol.htm](https://res.jzyx.com/passport/protocol.html) |          |
| 购买与货币             | IAP价格最高为Tier60，对应人民币648元（99美元），超过该数目会被拒。另外，如果出现大额的支付购买，只能一个一个的支付购买，不能一次性的购买。 | 1、这个规则有点无厘头，苹果自己设置的IAP最高是Tier87，对应到6498人民币，但是不明白为什么不让超过Tier60（可以是Tier60），大家按照要求来就行。<br />2、比如玩家需要购买6480元人民币的道具，只能把Tier60的道具购买10次，而不能设计为输入10，一次购买6480元的道具。 |          |
| 购买与货币             | 月卡需要按照苹果的要求操作。                                 | 方案一：游戏内微信/手Q/游客模式中都没有月卡内容，IAP没有月卡内容。（由于目前我们无法做到数据继承，所以强烈建议大家使用方案一）<br />方案二：如果无法做到数据继承，请不要设置月卡的IAP，将月卡购买逻辑设计成“人民币购买钻石，钻石在游戏内购买月卡”，可以参考天天飞车的贵族特权。<br />方案三：如果月卡是一个IAP，需要用人民币直接购买，请将月卡属性设置为non-renewing<br/>subscription型，且保证游客模式的数据可以继承至微信/手Q帐号体系中去；并且游客购买月卡后，在更换设备时，月卡还能继承使用。<br />特别注意，这里的“月卡”是指用需要人民币支付的IAP，购买后用户可以在一定时间段内，分阶段获取虚拟资产，所以是包括月卡、周卡、次数卡等我们创新的商业化道具的。 |          |
|                        | 如果有提交新的IAP，请务必在后台开放这个IAP给苹果审核。       | 1、苹果修改了规则，已上架的版本可以单独提交IAP给苹果审核，2015年12月上旬有项目有做尝试，1天过审IAP；<br />2、如果有提交新的IAP，请务必在后台开放这个IAP给苹果审核。IAP通过苹果审核后，开放与否，是我们控制的，不是苹果。 |          |
| 隐私                   | 要求用户提供个人信息，如邮箱地址，生日等，才能使用其功能的应用会被拒。 | 1、比如通过QQ登录游戏后，在提交IAP支付时，再次要求用户登录QQ才能支付的行为，会被苹果认为是在收集用户信息。<br />2、另外一点，没有guest登录模式，也会被这个理由拒绝。 |          |
| 用户界面               | 应用必须遵守Apple   iOS Human Interface Guidelines中的所有条款和条件。 | 比如：1、苹果默认的交互界面是不允许修改的，在状态提示栏加入自定义操作菜单导致被拒。<br />2、修改标准开关标准功能，例如：音量增加／减少，响铃／震动的应用会被拒。 |          |
|                        | 游客登录的UI设计和退出要符合苹果要求。                       | 1、登录UI的文字设计，直接用"游客登录"，苹果不允许用"游客体验"，"游客试玩"等相关字眼。<br />2、玩家在从QQ/微信登录退出后，还能通过"游客登录"进入游戏。 |          |
|                        | UI中不可以增加"检查更新"，"软件更新"等设计。                 | 苹果要求版本的更新必须在App Store内实现，App<br/>Store本身已经有更新管理机制，App出现更新检查等会被认为是影响用户体验，如果当前用户的App版本不是最新版本，引导用户更新到最新版本的提示是允许的。比如：<br />1）游戏启动后的资源包更新，是允许的；<br />2）进游戏后提示有新版本，点击跳转到App Store下载更新，是允许的；<br />3）在游戏设置里面的“关于XX"等标题栏中，用信息提示用户不是最新版本，是允许的。 |          |
| 其它                   | 除了上线前测试保证审核版本的质量外，保证审核期间后台环境的稳定性也是很重要的因素。 | 1、提审后服务器最好不要再更新或者做其他操作，我们很多次被苹果打回都是因为更新了服务器。<br />2、未开放对外网的IP连接权限会造成用户无法登录的问题，需注意在审核前进行确认（比如因为版权服务器限制了美国的IP，使美国服务器获取不到国内数据。）如下的测试方法供参考：<br />1）、手机连美国VPN（若没有美国VPN需先进行购买）<br />2）、手机浏览器上网搜“看IP”，查看自己手机的IP是否是美国IP<br />3）、玩游戏，进入评审服<br />4）、服务器开发同学查看后台日志，是否是美国IP进入<br />3、有项目组凌晨例行维护，屏蔽了正式服入口的功能，审核服入口使用了正式服务器的入口来做跳转，导致苹果链接不上服务器，提示服务器在做例行维护，请注意这点。<br />4、苹果审核中，提交新版本到预审，导致审核服务器环境变更，苹果无法登陆。 |          |
|                        | 因为支付环境不同步、参数配置错误等，可能会导致用户无法支付等问题，建议在提交审核前对环境、系统的配置进行检测。 | 1、注意支付要切换到正式环境，配置好相关现网支付信息并同步。<br />2、如果应用接入了Midas，一定要在提审前同步现网配置，并进行现网测试。<br />3、确认在iOS系统下，wifi和蜂窝网络环境可以正常进行支付。<br />具体方法是：发布证书打包后，用RDM上企业证书生成链接，生成企业证书，之后用沙箱帐号测试即可。注意要用发布证书打出来的包，上传后更改证书，而不是直接通过编译机再编译出来一个企业证书。<br />4、确保RDM版本号与安装包Version号一致。否则会被封控拦截，而无法支付。 |          |
|                        | 未获得用户初次同意便发送推送通知的程序将会被拒绝。           |                                                              |          |
| 法律要求               | 在App的描述中，如果有抽奖等活动，要在描述中增加一切抽奖活动与苹果公司无关的文字 | 在产品描述中，需要增加类似的说明，<br/>否则可能被苹果打回。如下是《欢乐斗牛》的描述，可以借鉴：<br />在游戏《欢乐斗牛》中，用户登陆时可以选择“微信登陆/QQ登陆/游客登陆”，三种登陆方式在iOS设备上的游戏数据不互通（包括等级、金币、钻石等）。用户在游戏中购买的游戏代币“钻石”仅限于在本应用中使用。<br />《欢乐斗牛》内一切抽奖活动与苹果公司无关，最终解释权归极致游戏所有。 |          |
| 竞赛，赌博，彩票和抽奖 | 抽奖等活动应当描述清楚活动主办方、奖品、发放方式，并且不能使用苹果或安卓产品作为奖品。 | 比如最终解释权归极致游戏所有；在奖品中不能出现苹果公司相关任何产品。不能出现安卓相关产品。 |          |
| 商标权与商标外观       | 应用名称中出现错误的Apple产品拼写（如，GPS for IPhone，<br/>iTunz）的应用会被拒。 | 界面iOS写为ios会被拒。                                       |          |
|                        | 商标，版权，专利等，如果涉及到版权，必须提供授权证明才能使用 | 如果有授权证明，在提交苹果审核的时候提供，因为提交审核时不能增加附件，可以在Review<br/>Notes以链接的形式提供给苹果。 |          |
| 元数据                 | 应用在提交审核过程中，所有包含到的URL都要处于正常运行状态，例如保密协议，相关支持页面等。 | 提交给苹果审核的各种URL必须保证能正常访问，而且内容要正确完整。 |          |
|                        | 描述中提到与应用内容和功能无关信息会被拒                     | 比如提供的截图，与应用无关或未能反映游戏的亮点玩法，可能会被苹果拒绝。苹果在屏幕截图审核越来越严格，后续提交屏幕截图时一定要提交最能表达游戏亮点玩法的图片。比如，赛车游戏，提供的图片要有赛车，赛道等，COC的游戏，要有战争场景，不要把市场宣传用的图片拿来提审。如果能提交预览视频更好。 |          |
|                        | App的评级要和产品相匹配                                      | 苹果公司加强了对评级的审核，以前评级没有问题的项目，很有可能因为评级问题，在新版本被打回，要求修改<br />App的评级目前有4+，9+，12+和17+，通过商务了解到，App里有露胸妹纸的，评级一律用17+ |          |
|                        | [提审的时候需要在Privacy Policy   URL项填写链接，参考 https://res.jzyx.com/passport/protocol.html](https://res.jzyx.com/passport/protocol.html) | [这个规则是苹果在2014年9月下旬新增加，以前是可选项，现在必选，IEG统一使用https://res.jzyx.com/passport/protocol.html](https://res.jzyx.com/passport/protocol.html) |          |
